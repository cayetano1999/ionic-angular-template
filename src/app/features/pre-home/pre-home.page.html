Ionic Capacitor Template For Angular Projects v1.0.0 Detalles del Template
  • Descripción Breve

    Este template es un punto de partida para proyectos de Angular con Ionic y Capacitor. Incluye una estructura de carpetas y archivos básicos para comenzar a desarrollar.

  • Características Principales

    Tecnologías Modernas

    Desarrollado con Angular 18 para aprovechar las últimas capacidades de este framework.

    Basado en Ionic Framework 8, ofreciendo un diseño moderno y adaptable.

    Soporte Multiplataforma

    Compatible con Android e iOS gracias a Capacitor 7.

    Soporte para desarrollo en navegadores con capacidades PWA.

    Plugins Capacitor Integrados

    • @capacitor/preferences: Almacenamiento seguro de datos.

    • @capacitor/network: Detección del estado de la red.

    • @capacitor/status-bar y @capacitor/haptics: Mejoras en la experiencia del usuario.

    • @capacitor-community/admob: Integración con anuncios publicitarios de AdMob.

    • @capacitor/browser: Navegación integrada desde la app.

    Integraciones Avanzadas

    Firebase: Soporte completo para autenticación, base de datos y almacenamiento.

    CryptoJS: Encriptación de datos para mayor seguridad.

    Optimización y Animaciones

    animation.css: Estilo y animaciones predefinidas para mejorar la interfaz de usuario.

    Swiper: Componente de carrusel dinámico y personalizable.

    Calidad del Código

    Configuración con ESLint y Prettier para mantener un código limpio y estructurado.

    Soporte de pruebas unitarias con Karma y Jasmine.

    Desempeño y Rendimiento

    Uso de lazy loading para una carga más eficiente de los módulos.

    RxJS para una programación reactiva y optimización del manejo de datos.

    Herramientas para Desarrollo

    Ionic Lab: Herramienta de vista previa para múltiples plataformas.

    Comandos predefinidos para iniciar, construir y testear el proyecto.

  • Servicios

    AdmobService

    El AdmobService es un servicio encargado de manejar la integración con Google AdMob utilizando el plugin de la comunidad de Capacitor. Este servicio permite gestionar anuncios como banners, intersticiales y recompensas dentro de la aplicación, con una configuración dinámica basada en remoteConfig.

    API Service

    El ApiService es un servicio genérico encargado de manejar las solicitudes HTTP hacia una API REST. Este servicio facilita las operaciones básicas de CRUD (Create, Read, Update, Delete) y permite la reutilización del código para interactuar con diferentes endpoints de la API.

    AuthService

    El AuthService es el servicio encargado de manejar la autenticación de usuarios en la aplicación. Se encarga de realizar el inicio de sesión (login), gestionar el token de autenticación y verificar su validez, utilizando ApiService para la comunicación con la API y StorageHelper para almacenar el token de sesión de forma segura.

    BackButtonService

    El BackButtonService gestiona el comportamiento del botón de retroceso en la aplicación, asegurando una navegación fluida y controlada. Este servicio permite cerrar modales abiertos, gestionar el historial de navegación y manejar la salida de la aplicación de manera segura.

    AppInBrowserService

    El AppInBrowserService permite abrir y cerrar enlaces externos dentro de un navegador in-app utilizando el plugin capacitor/browser. Esto mejora la experiencia del usuario al proporcionar una navegación fluida sin salir de la aplicación.

    CommunicationService

    El CommunicationService es un servicio que permite la comunicación entre componentes mediante un Subject de RxJS. Esto facilita la transferencia de datos sin necesidad de una relación padre-hijo directa, mejorando la modularidad de la aplicación.

    Firebase

    Se utilizan diferentes módulos de Firebase para manejar autenticación, almacenamiento, base de datos en tiempo real entre otros

    • FirebaseAnalyticsService: permite realizar un seguimiento de la actividad de los usuarios en la aplicación mediante Google Analytics para Firebase.

    • FirebaseDatabaseService: es el servicio encargado de interactuar con Firebase Realtime Database, permitiendo realizar operaciones CRUD (Crear, Leer, Actualizar y Eliminar) de manera eficiente. Este servicio facilita la gestión de datos en tiempo real dentro de la aplicación.

    • FirebaseRemoteConfigService: permite la configuración dinámica de la aplicación sin necesidad de lanzar actualizaciones en la tienda. Mediante Firebase Remote Config, la aplicación puede obtener valores en la nube y aplicarlos en tiempo real, optimizando la experiencia del usuario y permitiendo cambios instantáneos en la configuración. usuario.

    • FirebaseStorageService permite la gestión de archivos en Firebase Storage, permitiendo a la aplicación subir, eliminar y listar archivos almacenados en la nube. Este servicio facilita el almacenamiento de imágenes, documentos y otros recursos de manera eficiente y segura.

    • FirebaseAppService es responsable de inicializar la aplicación de Firebase en la app, asegurando que todos los servicios de Firebase (como Firestore, Auth, Storage, Remote Config, y Analytics) estén correctamente configurados antes de ser utilizados.
  • Guards

    AppMaintenanceGuard

    es un guard de rutas que restringe el acceso a la aplicación cuando está en modo mantenimiento. Si la variable APP_MAINTENANCE en el environment está activada, el usuario será redirigido a una página de mantenimiento en lugar de acceder a la app.

    ForceUpgradeGuard

    se encarga de verificar si la versión actual de la aplicación es inferior a la versión mínima permitida definida en remoteConfig.FEATURE_FLAGS.FORCE_UPDATE.active. Si la versión de la app está desactualizada, bloquea el acceso y muestra una alerta para forzar la actualización.

    InternetConnectionGuard

    se encarga de verificar si el dispositivo tiene conexión a Internet antes de permitir el acceso a ciertas rutas de la aplicación. Si no hay conexión, redirige al usuario a una pantalla específica (RoutesApp.NO_INTERNET) para informarle sobre la falta de conectividad.

    OnboardingGuard

    erifica si el usuario ha completado el onboarding antes de permitirle acceder a ciertas rutas de la aplicación. Si el usuario no ha finalizado el proceso de bienvenida, será redirigido a la pantalla correspondiente.
  • Helpers

    EncryptionHelper

    es un helper que proporciona métodos para encriptar y desencriptar texto usando AES (Advanced Encryption Standard) a través de la librería CryptoJS. Este helper es útil para proteger información sensible en la aplicación, como tokens, contraseñas o datos de usuario.

    StatusBarHelper

    es un helper que proporciona métodos para almacenar, recuperar y eliminar datos de forma segura en el almacenamiento local del dispositivo utilizando Capacitor Preferences. Además, emplea cifrado AES con EncryptionHelper para proteger la información almacenada.

    StorageHelper

    es un helper que proporciona métodos para almacenar, recuperar y eliminar datos de forma segura en el almacenamiento local del dispositivo utilizando Capacitor Preferences. Además, emplea cifrado AES con EncryptionHelper para proteger la información almacenada.
  • Interceptores

    HttpInterceptor

    es un interceptor de HTTP que se encarga de: ✔ Agregar automáticamente el token de autenticación a cada solicitud HTTP. ✔ Manejar errores de autenticación (401 Unauthorized), permitiendo acciones como el cierre de sesión o la obtención de un nuevo token.
  • Pipes

    ImageLoaderPipe

    El ImageLoaderPipe es un pipe personalizado que se encarga de procesar URLs de imágenes para determinar si son externas o internas. ✔ Si la imagen es externa (tiene http:// o https://), la devuelve sin cambios. ✔ Si la imagen es interna (sin URL completa), la busca en la carpeta assets/img/. Este pipe es útil para manejar imágenes en la aplicación sin necesidad de verificar manualmente la ruta en cada componente.

    SafeHtmlPipe

    es un pipe personalizado que permite sanitizar contenido antes de renderizarlo en la aplicación. Utiliza el servicio DomSanitizer de Angular para evitar que el navegador bloquee contenido considerado inseguro como HTML dinámico, estilos en línea, scripts y URLs.
